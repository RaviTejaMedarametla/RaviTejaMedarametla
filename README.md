## Research Direction

This portfolio focuses on hardware-aware machine learning systems designed to operate under real-world constraints such as memory limits, latency requirements, and compute budgets.

Across these projects, the primary workflow follows a systems pipeline:

**Data Engineering → Model Training → Model Compression → Hardware Optimization → Edge Deployment**

## Core Project Areas

### Edge AI Optimization

- **edge-ai-hardware-optimization**  
  Design space exploration for CNN pruning, quantization, and hardware trade-offs.

### ML Systems

- **neural-network-from-scratch**  
  Neural network implementation from first principles with configurable precision and hardware constraints.

- **classification-of-handwritten-digits1**  
  Benchmarking classical ML models with compression and resource-constrained evaluation.

### Data Engineering

- **data-analysis-for-hospitals**  
  Healthcare analytics pipeline with deterministic preprocessing and CPU-optimized inference.

- **nba-data-preprocessing**  
  Streaming data preprocessing pipeline with memory-aware execution and hardware telemetry.

### Production ML Systems

- **Data-Science-AI-Portfolio**  
  End-to-end ML lifecycle including training pipelines, ONNX optimization, FastAPI inference services, and drift monitoring.
